## Design Context

### Users
Returning travelers who tracked a trip on Polarsteps and want a print-ready
photo album. They are not designers. The tool should feel effortless: upload a
ZIP, get a beautiful album, tweak if needed, export PDF.

### Brand Personality
Clean, confident, precise. The interface is a professional print tool that stays
out of the way so the album content is the star.

### Aesthetic Direction
- Minimal chrome, generous whitespace, typography-forward. Photos and maps dominate.
- Professional and restrained. Avoid scrapbook/cutesy styling, stickers,
  gratuitous gradients, and decoration for its own sake.
- Dark and light themes must both feel considered.
- Print output is A4 landscape (297x210mm), with mm units for photo gaps.

### Design Principles
1. **Content is king** - the user's photos and maps are the product. UI recedes.
2. **Print-first fidelity** - if it cannot print correctly, it does not belong.
3. **Quiet confidence** - precise spacing, deliberate typography, restrained color.
4. **Inclusive by default** - WCAG AA for UI text and controls, keyboard navigation,
   reduced-motion support, RTL/i18n. Decorative album content is exempt from
   strict contrast ratios.
5. **One way to do it** - use existing design tokens, semantic CSS var names, rem
   units except 1px hairlines/outlines/optical nudges, and mm for print gaps.
