## Design Context

### Users
Returning travelers who tracked a trip on Polarsteps and want a print-ready photo album. They arrive in a nostalgic, reflective mindset - the trip is over and they want to preserve it. The tool must feel effortless: upload a ZIP, get a beautiful album, tweak if needed, export PDF. They are not designers.

### Brand Personality
Clean, confident, precise. The interface is a professional print tool - it stays out of the way so the album content is the star. No whimsy, no clutter. Every element earns its place.

### Aesthetic Direction
- **Visual tone**: Minimal chrome, generous whitespace, typography-forward. Photos dominate.
- **Typography**: Assistant (variable weight, native Hebrew) for UI + album headings. Frank Ruhl Libre (variable weight, native Hebrew) for album body text (serif option). Per-album `body_font` toggle between serif (Frank Ruhl Libre) and sans (Assistant). JetBrains Mono for coordinates and technical display.
- **Color**: Primary `#0063D1` (blue). Dark theme: deep blue-gray (`#1e1e2e`). Light theme: neutral gray (`#f3f4f6`). Semantic CSS vars (`--text`, `--bg`, `--surface`), Quasar accent via `--q-primary`.
- **Theme**: Auto dark/light mode. Both must feel equally considered - dark is not an afterthought.
- **Logo**: Blue open book with white compass needle, tilted ~25 degrees.
- **Anti-references**: Scrapbook-y, cutesy, or over-decorated. No stickers, no rounded-everything, no gratuitous gradients.
- **Print output**: A4 landscape (297x210mm), mm units for photo gaps. Must respect Chromium PDF constraints (no blur, box-shadow, blend modes, backdrop-filter).

### Design Principles
1. **Content is king** - the user's photos and maps are the product. UI recedes; album pages shine.
2. **Print-first fidelity** - every visual decision must hold up in a physical printed book. If it can't be printed, it doesn't belong.
3. **Quiet confidence** - precise spacing, deliberate typography, restrained color. No decoration for its own sake.
4. **Inclusive by default** - pragmatic WCAG AA: 4.5:1 contrast for text and interactive elements, keyboard navigation, reduced-motion support, RTL/i18n. Decorative album content is exempt from strict contrast ratios.
5. **One way to do it** - consistent use of design tokens. No ad-hoc values when a token exists. Semantic CSS var names, rem units (never px), mm for print gaps.
