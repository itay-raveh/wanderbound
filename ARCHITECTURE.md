## What This App Does

Converts a Polarsteps data export ZIP into a printable photo album (PDF). Users upload their ZIP, the backend processes GPS tracks / weather / elevations / photos, and the frontend renders an interactive album editor that can be exported to PDF via headless Playwright.

## Data Flow

### Upload & Processing

```
User uploads ZIP â†’ POST /users â†’ extract ZIP to data/users/{uid}/
  â†’ parse user.json â†’ discover trip directories â†’ save User to DB
  â†’ return UserCreated { user, trips[] }

GET /users/process (SSE) â†’ ProcessingSession (background task, reconnectable)
  Per trip:
    1. elevations  â€” Open-Meteo DEM API (batched, 100/request)
    2. weather     â€” Open-Meteo Archive API (1 call/step, concurrent)
    3. layouts     â€” read photos/videos from step dirs, build page layouts
       â†’ flatten media to trip root dir
       â†’ download cover photo from Polarsteps CDN
    4. frames      â€” ffmpeg extract poster from each .mp4 (semaphore-limited)
    5. thumbs      â€” Pillow generate WebP thumbnails at 200px + 800px
  â†’ build_segments (Polars pipeline: GPS â†’ typed segments)
  â†’ bulk insert Album + Steps + Segments to DB
```

### PDF Export

```
POST /albums/{aid}/pdf
  â†’ Playwright opens /print/{aid} on frontend
  â†’ PrintView loads album, waits for all images + fonts
  â†’ sets window.__PRINT_READY__ = true
  â†’ Playwright captures PDF (A4 landscape, CSS page size, print background)
```

### Album Editing

```
EditorView â†’ useAlbumQuery + useAlbumDataQuery â†’ AlbumViewer
  â†’ AlbumToolbar: PATCH /albums/{aid} (title, subtitle, covers, ranges)
  â†’ StepEntry: drag-and-drop photos, inline name/description editing â†’ PATCH /albums/{aid}/steps/{sid}
  â†’ CoverPage: inline title/subtitle editing â†’ PATCH /albums/{aid}
  â†’ All mutations use optimistic updates (Pinia Colada cache)
```

## Auth Model

Cookie-based, no encryption. The `uid` cookie holds the Polarsteps user ID (integer). `UserDep` in `deps.py` reads it and loads the User from DB. No sessions, no tokens, no passwords. Pre-alpha only.

## GPS Segmentation Pipeline (`segments.py`)

5-stage Polars pipeline, all vectorized:

1. **Ingest** â€” merge step waypoints + GPS locations, dedup, remove teleports/spikes, densify slow edges to ~15m
2. **Label** â€” classify edges: flight (>200 km/h), hike (<6.5 km/h within 4h gap), other
3. **Absorb** â€” fold noise gaps (<3h, <4km), overnight camps (<1km, <20h), GPS blackouts back into hikes
4. **Validate** â€” downgrade undersized hikes (<2h, <2km, <1km displacement) and short flights (<100km) to "other"
5. **Emit** â€” RDP simplify, resolve "other" â†’ walking/driving by avg speed, stitch consecutive segments

## Photo Layout Algorithm (`builder.py`)

Packs step photos into album pages:
- Split into portraits (aspect â‰¤ 4:5) and landscapes
- Cover = best portrait (closest to 4:5) or first landscape
- Mixed pages: 1 portrait + 2 landscapes (minimize total page count)
- Portrait pages: batched by 3
- Landscape pages: optimal 4/3 decomposition
- Frontend renders grids in StepPhotoPage with orientation-aware CSS layouts
