# Frontend

See @mise.toml — relevant tasks: `dev:frontend`, `test:frontend`, `test:e2e`, `lint:frontend`, `format:frontend`, `build`.
To regenerate the OpenAPI client: `bun x openapi-ts` (no mise task).

## Non-Obvious Patterns

- `useWindowVirtualizer`: custom virtualizer replacing @tanstack/vue-virtual due to Vue 3.4 computed dedup bug.
- `useAlbum()`: provides album context via inject — must be called inside AlbumViewer hierarchy.
- `useTextMeasure()`: hidden DOM containers measure description text to determine layout (short/long/extra-long).
- Pinia Colada mutations: optimistic cache update → push undo stack → revert on error → invalidate on settle.
- Map routing: Map Matching API for dense GPS traces (<2km avg spacing), Directions API for sparse. Chunked with in-memory cache.
- `stripPhotos()`: atomically removes photos from all page lists to prevent duplicates during drag-and-drop.
- Font constants (`ALLOWED_FONTS`, defaults, fallback stacks) live in `src/utils/fonts.ts` — single source of truth.
- `useTextMeasure()` containers live in `document.body`, outside `.album-container` — they only see `:root` CSS vars, not inline style overrides.
- Quasar q-select `#option` slots: use `v-bind="itemProps"` for click handling, never `toggleOption`.

## CSS Rules

- Use rem, never px. Photo gaps use mm units for print accuracy.
- Use semantic CSS var names (--bg, --text, --surface), never --album-* prefix.
- Use --q-primary for accent color (shares value with Quasar components).
- Design tokens are in App.vue `:root`. Dark/light mode key: `"album-dark-mode"` in localStorage.
- Type scale: `--type-xs` (0.75rem) is the smallest UI size. `--type-3xs` (0.5625rem) is print-only (album pages at A4 scale).

## Do NOT

- Edit `src/client/` — generated by @hey-api/openapi-ts
- Use px — use rem
- Use --accent — use --q-primary
- Use default exports
